# Prompt for Google Stitch (Frontend Generation)

You can copy and paste the following prompt into Google Stitch (or your preferred AI frontend generation tool) to create a premium, modern UI for the AI Restaurant Recommender system.

***

**System Role & Context**
You are an expert Frontend Developer and UI/UX Designer. Your task is to generate the frontend UI and necessary images/assets for an AI-powered Restaurant Recommendation engine. The backend is already built, and we need a stunning presentation layer to interact with the API. The frontend will be served using Node.js.

**Design & Aesthetics Requirements**
1. **Rich Aesthetics:** Create a visually stunning, premium design that "wows" users at first glance. 
2. **Modern Styling:** Use a sleek dark mode theme with vibrant accent colors (e.g., deep purples, glowing neon blues, or warm fiery oranges to represent food). Incorporate glassmorphism effects for cards and modals.
3. **Typography:** Use modern, highly readable web fonts like Inter, Roboto, or Outfit. Avoid default browser fonts.
4. **Dynamic & Interactive:** Ensure the design feels alive. Add smooth hover effects, micro-animations on buttons and cards, and sleek transitions between states (e.g., loading restaurant recommendations).
5. **No Placeholders:** Generate realistic, high-quality sample images or illustrations for restaurants instead of using generic gray placeholders.

**Technical Stack**
- **Framework:** Node.js (used to serve the frontend).
- **Structure:** Semantic HTML5.
- **Styling:** Vanilla CSS (or modern CSS features) to ensure maximum flexibility and control. Avoid generic frameworks unless absolutely necessary.
- **Logic:** Vanilla JavaScript for handling API calls, user interactions, and dynamic DOM updates.

**Core Pages/Components Needed**
1. **Landing Page:** A hero section with a compelling headline, an input area where users can describe what they want to eat (e.g., "A cozy Italian place for a date night under $50"), and a prominent "Find Restaurants" call-to-action button.
2. **Loading State:** A dynamic, premium loading animation that displays while the AI is computing the recommendations.
3. **Results View (Ranking):** A list or grid of restaurant cards. Each card must display:
   - High-quality restaurant image.
   - Restaurant name, cuisine tags, and price rating.
   - An "AI Match Score" (e.g., 95% Match) prominently displayed.
   - A short, AI-generated summary of why this restaurant is recommended based on the user's prompt.
4. **Detail Modal:** A glassmorphism modal that opens when a card is clicked, showing more details, reviews, and a map placeholder.

**Actionable Request**
Please generate the complete HTML, CSS, and Client-side JavaScript code for these views, along with the Node.js (Express) server setup needed to serve them. Also, provide the assets/images or the code to generate/fetch premium placeholder images. Ensure the final result is fully responsive and adheres to modern SEO best practices (semantic tags, proper heading hierarchy).
